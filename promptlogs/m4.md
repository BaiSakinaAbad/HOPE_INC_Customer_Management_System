This is a great, clean UI foundation. To integrate Supabase authentication (email/password and Google OAuth) while keeping your codebase maintainable, we need to separate the authentication logic from the UI components.

Here is the step-by-step process to wire up your UI to Supabase.

#### **Step 1:** Initialize the Supabase Client

- Created `.env.local` and stored environmental variables
- Create a new file `src/lib/supabase.ts`

#### **Step 2:** Create an Auth Context

- Create `src/components/AuthContext.tsx:`
- Wrap your `AuthProvider` in App.tsx

#### **Step 3:** Update RegisterPage.tsx

Update the state and `handleSubmit` in `RegisterPage.tsx`

#### **Step 4:** Update LoginPage.tsx

Update the state and `handleSubmit` in `LoginPage.tsx`

#### **Step 5:** Wire up Google OAuth

Update `GoogleButton` in `components/composites/AuthComposites.tsx`

## **What I changed / improved:**

- installed supabase package
- created lib/supabase.ts for supabase utilisation (backend)
- AuthComposites.tsx: integrated Supabase client to the 'continue with Google' button so that it proceeds to OAuth
- LoginPage.tsx/RegisterPage.tsx: wired input fields, and connected the form submission to Supabase's email and password authentication method.

## **Why:**

- We need a single, shared instance of the Supabase client to interact with the backend. Creating it in a dedicated file prevents redundant network calls and keeps your configuration centralized.
- Authentication state (like whether a user is logged in or not) needs to be accessed globally. An Auth Context prevents "prop drilling" and gives you a clean way to protect routes later.
- We need to capture the form data and pass it to Supabase. Supabase's `signUp` function accepts a `user_metadata` object, which is perfect for storing the extra fields (First Name, Last Name, Username) you have in your UI.
- Similar to registration, we need to bind the inputs to React state and call `supabase.auth.signInWithPassword`.
- To make the `GoogleButton` functional, you just need to call `signInWithOAuth`. Make sure you have configured Google as an Auth Provider inside your Supabase dashboard first.
