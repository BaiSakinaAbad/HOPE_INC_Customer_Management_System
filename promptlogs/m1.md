### Entry 2
**Date:** March 28, 2026

**Task:** Automate Changelog Updates

**Prompt given to AI:** "Create a GitHub Action that triggers on a PR merge to the dev branch. It should extract text from the PR description between ### LOG_START and ### LOG_END and append it to changelog.md in the all/docs branch."

**What the AI produced:** A YAML workflow file using `sed` to inject text into the markdown file.

**What I changed / improved:** Added a specific HTML comment marker (``) in the markdown file to act as a precise anchor for the injection.

**Why:** To prevent the AI from accidentally overwriting headers or appending to the very bottom of the file.

**What I learned / decided:** GitHub Actions require "Read and Write" permissions to be manually enabled in repository settings to push changes back to a branch.

### Entry 3

**Task:** Customer List Rights Gating

**Prompt given to AI:** "Create a Supabase service function to fetch customers. If the user is 'USER', filter for 'ACTIVE'. If 'SUPERADMIN', show all. Also, create a React table that hides the 'Stamp' column for 'USER' roles."

**Why:** To resolve a "Table not found" schema cache error.

**What I learned / decided:** Always verify database naming conventions (singular vs. plural) early, as standard AI prompts often assume singular naming.
